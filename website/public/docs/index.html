Complete guide to AI Code Terminal

Introduction

AI Code Terminal is a production-ready, single-tenant development environment that provides god-mode shell access through your browser. Built with security and simplicity in mind, it offers a complete terminal interface for a single authorized GitHub user to work directly in their repositories.

Key Features

  • God-mode terminal access: Full bash/shell access in browser
  • Single-tenant authentication: GitHub OAuth for one authorized user
  • Direct Claude integration: Run claude login directly in terminal
  • Repository workspace management: Clone and work in GitHub repositories
  • Real-time terminal I/O: Powered by xterm.js + node-pty
  • Security hardening: Containerized with resource limits

Quick Start

Get up and running with AI Code Terminal in just a few steps:

1. Clone the Repository

git clone https://github.com/your-username/ai-coding-terminal.git
cd ai-coding-terminal

2. Set Up Environment

cp env.example .env
# Edit .env with your GitHub OAuth credentials

3. Run with Docker

npm run docker:build
npm run docker:run

4. Access Your Terminal

Open http://localhost:3014 in your browser and log in with GitHub.

Installation

Prerequisites

  • Node.js 18+ and npm
  • Docker and Docker Compose
  • GitHub OAuth Application

GitHub OAuth Setup

Before installing, you need to create a GitHub OAuth application:

1. Create OAuth App

Go to GitHub Settings → Developer settings → OAuth Apps → New OAuth App

2. Configure Settings

  • Application name: AI Code Terminal
  • Homepage URL: http://localhost:3014
  • Authorization callback URL: http://localhost:3014/auth/github/callback

Local Development

# Install dependencies
npm install

# Set up environment
cp env.example .env

# Initialize database
npx prisma generate
npx prisma db push

# Start development server
npm run dev

Configuration

Configure your AI Code Terminal instance using environment variables in the .env file:

Required Variables

Variable Description Example
GITHUB_CLIENT_ID GitHub OAuth application client ID abc123def456
GITHUB_CLIENT_SECRET GitHub OAuth application client secret secret_key_here
TENANT_GITHUB_USERNAME Authorized GitHub username (single-tenant) your-username
JWT_SECRET JWT signing secret (32+ characters) your-super-secret-jwt-key-here

Optional Variables

# Server configuration
PORT=3014
NODE_ENV=production

# Database
DATABASE_URL=file:./data/database.db

# Workspace limits
MAX_WORKSPACES_PER_USER=10
WORKSPACE_CLEANUP_DAYS=30

# GitHub OAuth callback
GITHUB_CALLBACK_URL=http://localhost:3014/auth/github/callback

Terminal Access

The AI Code Terminal provides full shell access through your browser with real-time I/O.

Terminal Features

  • Full shell access: bash, zsh, or system default shell
  • Real-time I/O: Instant command execution and output
  • Terminal resizing: Responsive terminal that adapts to window size
  • Process isolation: Each session runs in isolated workspace
  • Resource limits: Memory and CPU limits for security

Workspace Management

Manage your GitHub repositories as isolated workspaces within the terminal environment.

Creating Workspaces

Workspaces are created automatically when you clone a repository through the web interface:

  1. Click "Browse Repositories" in the sidebar
  2. Select a repository from your GitHub account
  3. The repository is cloned to an isolated workspace directory
  4. Your terminal session automatically navigates to the workspace

Workspace Structure

workspaces/
├── my-project/              # Repository workspace
│   ├── .git/               # Git repository
│   ├── CLAUDE.md           # Auto-generated Claude instructions
│   └── [project files]    # Your project code
└── another-project/        # Another workspace
    └── [project files]

Workspace Limits

  • Maximum workspaces per user: configurable (default: 10)
  • Automatic cleanup: inactive workspaces cleaned after 30 days
  • Isolated environments: each workspace runs in its own directory

GitHub Integration

Seamless integration with GitHub for authentication and repository management.

Authentication Flow

  1. User clicks "Login with GitHub"
  2. GitHub OAuth authorization for the authorized tenant user
  3. Server validates user against TENANT_GITHUB_USERNAME
  4. JWT token generated and stored in localStorage
  5. User gains access to terminal and repositories

Repository Access

Access all your GitHub repositories:

  • Browse public and private repositories
  • Clone repositories directly to workspaces
  • Full Git operations (push, pull, commit, etc.)
  • SSH key management for secure Git operations

System Overview

Understanding the architecture of AI Code Terminal.

Core Components

Server Architecture

  • Express.js HTTP server with Socket.IO for real-time communication
  • Single-tenant GitHub OAuth authentication with JWT tokens
  • Terminal session management with node-pty for shell access
  • Direct shell spawning in isolated workspace directories

Database Management

  • Prisma ORM with SQLite for lightweight data storage
  • Settings table for encrypted GitHub OAuth tokens
  • Workspace table for repository management
  • Session table for terminal session tracking

Web Interface

  • Server-rendered EJS templates with Vue.js interactivity
  • Real-time terminal via Socket.IO and xterm.js
  • Two-panel layout: workspace sidebar + terminal panel
  • Repository browsing and cloning interface

Security Model

AI Code Terminal implements comprehensive security measures for safe operation.

Authentication & Authorization

  • Single-tenant access enforced via TENANT_GITHUB_USERNAME
  • JWT tokens with configurable expiration (default: 7 days)
  • GitHub OAuth tokens encrypted in database using AES-256-CBC
  • No server-side Claude token storage (user manages directly)

Container Security

  • Docker containers run as non-root user
  • Read-only root filesystem with writable tmpfs volumes
  • Resource limits enforced (memory, CPU)
  • Process isolation within workspace boundaries

Network Security

  • CORS configured for production/development environments
  • Input validation and sanitization on all endpoints
  • Comprehensive error handling and logging
  • Secure WebSocket connections for terminal I/O
